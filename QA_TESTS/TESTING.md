# Test cases:

TC-01 [Positive] Анализ по умолчанию
* Описание: Запуск программы без указания
    параметров (проверка значений по умолчанию).
* Предусловие: В папке со скриптом есть файл dates.log.
* Входные данные: python log_analyzer.py
* Ожидаемый результат: Программа берет dates.log, выводит статистику в консоль
    и создает report.txt.

TC-02 [Negative] Файл неверного формата
* Описание: Попытка проанализировать текстовый файл, который не является логом
    (например, report.txt).
* Входные данные: -f report.txt
* Ожидаемый результат: Программа не падает. Все строки, которые она не смогла
    распознать как уровни логов, записываются в категорию UNKNOWN.

TC-03 [Positive] Анализ с полной детализацией
* Описание: Использование флагов выбора файла и подробного вывода.
* Входные данные: -f dates.log -d
* Ожидаемый результат: В консоли отображается статистика И полный список
    всех строк логов.

TC-04 [Negative] Некорректный тип файла
* Описание: Подсовываем программе файл, который нельзя прочитать как текст
    (картинка .jpg, музыка .mp3).
* Ожидаемый результат: Выводится понятное сообщение об ошибке (например,
    «Невозможно прочитать файл») и завершается без системного «краша».

TC-05 [Boundary] Пустой файл
* Описание: Подсовываем пустой файл
* Ожидаемый результат: Выводится сообщение о том, что файл пустой

TC-06 [Boundary] 1000 логов
* Описание: Подсовываем большой файл с кол-вом 1000 логов
* Ожидаемый результат: Всё успешно обработается, как и с меньшим кол-вом логов

TC-07 [Negative] Несуществующий файл
* Описание: Подсовываем флагу -f несуществующий файл
* Ожидаемый результат: Выведется сообщение о том, что данный файл не найден
